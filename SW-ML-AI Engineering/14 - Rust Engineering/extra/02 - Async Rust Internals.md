# ⚡ Async Rust Internals

## Introduction

Rust's asynchronous programming model is built on zero‑cost futures and a pluggable runtime, allowing highly concurrent applications without the overhead of threads. The `async`/`await` syntax sugar hides a sophisticated state machine that integrates with the [[01 - Rust Memory Model Deep Dive|memory model]] to produce efficient, stack‑pinned futures. Understanding these internals is crucial for building custom executors, debugging performance issues, and avoiding pitfalls like self‑referential structs.

The async model is built on the `Future` trait, which represents a computation that may not be ready immediately. When you `.await` a future, control is yielded back to the executor, which can poll other tasks. This design enables millions of concurrent connections on a single thread, as demonstrated by production systems like [[03 - Rust Cryptography and Security|rustls]] and [[04 - Embedded Rust and IoT|embedded]] runtimes.

This deep dive explores the `Poll` state machine, waker mechanics, executor designs (single‑threaded vs. multi‑threaded work‑stealing), and the infamous `Pin`/`Unpin` API. We’ll also examine real‑world async runtimes and build a mini‑tokio to see how the pieces fit together.

## 1. Futures, Wakers, and Executors

A future in Rust is a value that may not be ready yet, represented by the `Future` trait:

```rust
pub trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}
```

### Poll States

- `Poll::Ready(T)`: The future has completed with value `T`.
- `Poll::Pending`: The future is not ready; the executor should try again later.

### Waker and Context

When a future returns `Poll::Pending`, it must arrange to be woken up when progress can be made. This is done via `cx.waker().wake_by_ref()`. The waker is a thread‑safe handle that tells the executor to re‑poll the task.

**Real case:** Tokio's waker implementation uses a linked list of tasks waiting on I/O events, enabling efficient notification without polling.

⚠️ **Warning:** Failing to register a waker causes the task to never be polled again (a "lost" task). Always call `cx.waker().wake_by_ref()` when the future cannot make progress.

💡 **Tip:** For I/O‑bound tasks, use `tokio::io::AsyncReadExt` which handles waker registration automatically. For CPU‑bound work, consider `tokio::task::spawn_blocking` to avoid blocking the async executor.

### Executor Designs

| Executor Type | Characteristics | Use Case |
|---------------|----------------|----------|
| Single‑threaded | Simple, no thread‑switching overhead | Embedded, WebAssembly, low‑latency |
| Multi‑threaded (work‑stealing) | Scales across cores, load‑balances tasks | High‑throughput servers, databases |
| Thread‑per‑core | Fixed threads, no synchronization | Network functions, real‑time systems |

**Real case:** Linkerd, the service mesh, uses a single‑threaded executor per core for predictable latency in its proxy.

## 2. Pin, Unpin, and Self‑Referential Futures

`Pin` is a wrapper that prevents a value from being moved in memory. This is crucial for self‑referential structs (e.g., a future that holds a reference to its own internal state). Most types are `Unpin`, meaning they can be safely moved even when pinned.

### Why Pin?

When a future is polled, it may store pointers to its own fields. If the future is moved after being polled, those pointers become invalid, leading to undefined behavior. `Pin` ensures that once a future is polled, it stays at the same memory address.

### Unpin Trait

`Unpin` is an auto trait implemented by types that can be safely moved. `Pin<&mut T>` only implements `DerefMut` if `T: Unpin`. For self‑referential futures (e.g., those generated by `async fn`), you must use `Pin::new_unchecked` or the `pin_mut!` macro.

⚠️ **Warning:** Calling `Pin::new_unchecked` on a type that is not `Unpin` and not actually pinned is unsound and can cause memory corruption. Use `Box::pin` or `tokio::pin!` for safe pinning.

💡 **Tip:** Most user‑written futures are `Unpin` automatically. Only worry about `Pin` when implementing `Future` manually with `&mut self` references.

## 3. Async Runtimes Comparison

Different runtimes offer different trade‑offs:

| Runtime | Threading Model | Overhead | Best For |
|---------|----------------|----------|----------|
| Tokio | Multi‑threaded work‑stealing | Medium | General‑purpose, high‑performance servers |
| async‑std | Multi‑threaded (smol‑based) | Medium | Ease of use, familiar API |
| smol | Multi‑threaded (pluggable) | Low | Lightweight, customizable |
| monoio | Thread‑per‑core | Low | Extreme performance, io_uring |
| embassy | Single‑threaded (embedded) | Very low | Microcontrollers, no‑std |

**Real case:** Cloudflare's proxy uses a custom runtime based on `tokio` with io_uring for reduced syscalls on Linux.

Formula: `Async_Overhead = Waker_Queue_Size + Task_Metadata`. Minimize overhead by batching waker notifications and reusing task slots.

## 4. Building a Mini‑Tokio

Let's implement a minimal async runtime with a single‑threaded executor and timer.

```rust
// mini_tokio.rs
use std::collections::VecDeque;
use std::future::Future;
use std::pin::Pin;
use std::sync::{Arc, Mutex};
use std::task::{Context, Poll, Waker};
use std::thread;
use std::time::{Duration, Instant};

type Task = Pin<Box<dyn Future<Output = ()> + Send>>;

struct MiniTokio {
    tasks: Mutex<VecDeque<Task>>,
    timer_queue: Mutex<Vec<(Instant, Waker)>>,
}

impl MiniTokio {
    fn new() -> Self {
        Self {
            tasks: Mutex::new(VecDeque::new()),
            timer_queue: Mutex::new(Vec::new()),
        }
    }

    fn spawn<F: Future<Output = ()> + Send + 'static>(&self, future: F) {
        self.tasks.lock().unwrap().push_back(Box::pin(future));
    }

    fn run(&self) {
        loop {
            // Poll all ready tasks
            let mut tasks = self.tasks.lock().unwrap();
            if let Some(task) = tasks.pop_front() {
                let waker = futures::task::noop_waker();
                let mut context = Context::from_waker(&waker);
                match task.poll(&mut context) {
                    Poll::Ready(()) => {}
                    Poll::Pending => tasks.push_back(task),
                }
            }
            drop(tasks);
            // Check timers
            let now = Instant::now();
            let mut timer_queue = self.timer_queue.lock().unwrap();
            timer_queue.retain(|(until, waker)| {
                if now >= *until {
                    waker.wake_by_ref();
                    false
                } else {
                    true
                }
            });
            drop(timer_queue);
            thread::sleep(Duration::from_millis(1));
        }
    }
}

async fn delay(duration: Duration) {
    let (tx, rx) = std::sync::mpsc::channel();
    let until = Instant::now() + duration;
    // In a real runtime, register with timer queue
    thread::spawn(move || {
        thread::sleep(duration);
        let _ = tx.send(());
    });
    let _ = rx.recv();
}

#[tokio::main]
async fn main() {
    let rt = Arc::new(MiniTokio::new());
    rt.spawn(async {
        println!("Task 1 started");
        delay(Duration::from_millis(100)).await;
        println!("Task 1 finished");
    });
    rt.spawn(async {
        println!("Task 2 started");
        delay(Duration::from_millis(50)).await;
        println!("Task 2 finished");
    });
    rt.run();
}
```

## 📦 Compression Code

Complete Rust script implementing a custom single‑threaded executor with timer wheel.

```rust
// timer_executor.rs
use std::collections::{BinaryHeap, VecDeque};
use std::cmp::Ordering;
use std::future::Future;
use std::pin::Pin;
use std::sync::{Arc, Mutex};
use std::task::{Context, Poll, Waker};
use std::time::{Duration, Instant};

struct TimerEntry {
    expiration: Instant,
    waker: Waker,
}

impl PartialEq for TimerEntry {
    fn eq(&self, other: &Self) -> bool {
        self.expiration == other.expiration
    }
}
impl Eq for TimerEntry {}
impl PartialOrd for TimerEntry {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}
impl Ord for TimerEntry {
    fn cmp(&self, other: &Self) -> Ordering {
        // Reverse for min‑heap
        other.expiration.cmp(&self.expiration)
    }
}

pub struct SimpleExecutor {
    tasks: Mutex<VecDeque<Pin<Box<dyn Future<Output = ()> + Send>>>>,
    timer_queue: Mutex<BinaryHeap<TimerEntry>>,
}

impl SimpleExecutor {
    pub fn new() -> Self {
        Self {
            tasks: Mutex::new(VecDeque::new()),
            timer_queue: Mutex::new(BinaryHeap::new()),
        }
    }

    pub fn spawn<F: Future<Output = ()> + Send + 'static>(&self, future: F) {
        self.tasks.lock().unwrap().push_back(Box::pin(future));
    }

    pub fn run(&self) {
        loop {
            let mut progress = false;
            // Poll tasks
            {
                let mut tasks = self.tasks.lock().unwrap();
                if let Some(mut task) = tasks.pop_front() {
                    let waker = futures::task::noop_waker();
                    let mut cx = Context::from_waker(&waker);
                    match task.as_mut().poll(&mut cx) {
                        Poll::Ready(()) => progress = true,
                        Poll::Pending => tasks.push_back(task),
                    }
                }
            }
            // Fire timers
            {
                let now = Instant::now();
                let mut queue = self.timer_queue.lock().unwrap();
                while let Some(entry) = queue.peek() {
                    if entry.expiration <= now {
                        let entry = queue.pop().unwrap();
                        entry.waker.wake_by_ref();
                        progress = true;
                    } else {
                        break;
                    }
                }
            }
            if !progress {
                std::thread::sleep(Duration::from_millis(1));
            }
        }
    }

    pub fn sleep(&self, duration: Duration, waker: Waker) {
        let entry = TimerEntry {
            expiration: Instant::now() + duration,
            waker,
        };
        self.timer_queue.lock().unwrap().push(entry);
    }
}

async fn demo(executor: Arc<SimpleExecutor>) {
    println!("Start");
    let (tx, rx) = std::sync::mpsc::channel();
    let waker = futures::task::noop_waker();
    executor.sleep(Duration::from_millis(50), waker);
    // Simulate async work
    std::thread::spawn(move || {
        std::thread::sleep(Duration::from_millis(50));
        let _ = tx.send(());
    });
    let _ = rx.recv();
    println!("Done");
}

fn main() {
    let exec = Arc::new(SimpleExecutor::new());
    exec.spawn(demo(exec.clone()));
    exec.run();
}
```
